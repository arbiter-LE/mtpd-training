Phase 0

At Signing — Day 0

STOP — do not start without all three: (1) service agreement signed (you confirm, not a doc), (2) the agency's own Supabase project provisioned, (3) legal citation registry re-verified — every PENDING cite the agency's content will rely on promoted to GOOD LAW first.

Email the chief the onboarding form URL

Phase 2

Supabase — Stand Up the Database

You run everything in the Supabase console; Claude generates the exact SQL. No DROP/DELETE/TRUNCATE without naming the project.

11
Create the Supabase project
Name arbiterle-<short>, region US East. Save the DB password in your manager.
Console
12
Copy the API credentials
Settings → API: Project URL and anon public key. You paste these into the registry later — Claude never invents or overwrites them.
Console
13
Add the service-role key to Vercel
Settings → API: copy the service_role key and set it as SUPABASE_SERVICE_KEY_<SUB> in Vercel env vars — never in source, never in the registry. /api/grade uses it to grade quizzes and write completion records server-side; without it, grading fails closed for this department.
Console
14
Run the schema SQL
Model on the EGPD init (_database/egpd/...init-schema.sql). File the new agency's copy under _database/<subdomain>/.
Claude
15
Run RLS hardening
Own-record + is_admin() model, anon access closed. Mirror the EGPD/MTPD RLS files. Verify anon cannot read officers/completions.
Claude
16
Add officers to Supabase (seed the roster)
Claude generates an INSERT … ON CONFLICT (badge_number) DO NOTHING block — admin for command staff, officer otherwise. You run it, then confirm the row count in the dashboard.
Claude
17
Link officers to auth users
Confirm each seeded officer row is auth-linked (auth_uid) so logins map to records.
Console
18
Configure Auth URLs (REQUIRED)
Authentication → URL Configuration. Site URL https://<sub>.arbiterle.com; redirect allow-list = that URL and …/?reset=1, nothing else. Skipping this dead-ends every password reset at localhost.
Console
19
Configure custom SMTP (Google Workspace)
smtp.gmail.com : 587, user andrew@arbiterle.com with a dedicated Google App Password for this project, sender andrew@arbiterle.com / "Arbiter LE". Never Resend — Microsoft/EOP junks its shared IPs (6/17 EGPD launch lesson). Send a test reset — confirm it lands on the subdomain, not localhost.
Console
Phase 3

Platform Config — Registry, Badge, Theme

All in the company project. Never fork shared code (app.js / config.js / index.html / styles.css) per agency.

20
Add the registry entry
js/departments/registry.js — one block: subdomain, names, badge path, supabaseUrl/supabaseKey (pasted exactly from step 12), scheduleStart, cadence (the agreed unlock/due rhythm — standard is 2 modules/month), moduleScripts, features. Triple-check the keys — mixing them corrupts officer data.
Claude
21
Add the badge image
assets/<subdomain>-badge.png — exact case (Vercel/Linux is case-sensitive).
Claude
22
Add the theme block
html[data-theme="<sub>"] token-override block in css/styles.css, using the agency's brand colors. EGPD (charcoal-green/gold) is the reference.
Claude
Phase 4

Module Content

Author in js/modules/<subdomain>/. Never edit or borrow from another agency's modules.

23
Author the modules
Verbatim policy · approved roads only · no real names · ≥ 8 quiz questions per module, each answerable solely from that module · only GOOD-LAW citations (no holding from memory).
Claude
24
Supervisor-track variants
If features.supervisorTrack is on, author the command-lens reading overlay + graded questions per module.
Claude
25
Build the server-side answer keys
node _dev/build-answer-keys.js — strips correct:/feedback: from the client module files and compiles the keys into api/grade.js. Answer keys never ship in client JS; the smoke test hard-fails if any do. Author modules with keys inline as always — this runs after.
Claude
26
Validate scenario structure
node _dev/validate-scenarios.js — clean before moving on.
Claude
Phase 5

27
Smoke test every department
node _dev/smoke-departments.js — builds each registered department, validates modules, blocks hardcoded subdomains, and hard-fails on any answer key left in client JS. Must pass.
Claude
28
